SOMMAIRE

  • • Windows Server, Active Directory, nom de domaine (Domain Controller / DC)
  • • Deux postes Windows 10 pour deux utilisateurs différents (postes clients dans la plage DHCP 10.10.10.0/24)
  • • Un serveur Debian VOIP avec Asterisk
  • • Un serveur Debian DNS avec Bind9
  • • Un pare-feu pfsense pour la DMZ (plusieurs zones DMZ avec serveurs exposés comme Proxy, Reverse Proxy, WordPress)
  • • Serveur de supervision Zabbix pour le monitoring
  • • Serveur RSyslog pour la centralisation des logs
  • • Serveur DHCP Kea (attribution des adresses 10.10.10.10 à 10.10.10.99)
  • • Reverse Proxy (RP) comme point d'entrée web
  • • Proxy(s) pour la gestion du trafic (forward ou reverse)
  • • Autres serveurs internes : Nextcloud, Surveillance, etc.
  • • Routeurs/Switches/Firewalls multiples (4) connectant les sous-réseaux (10.5.0.0, 10.75.0.0, 10.25.0.0, 10.30.0.0, etc.)
